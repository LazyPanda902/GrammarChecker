\# Security Policy



\## Supported Versions



This project is currently under active development, so security fixes will be applied to the latest version in the `main` branch.



\## Reporting a Vulnerability



If you discover a security issue, please report it responsibly by contacting the repository owner through GitHub.



When reporting an issue, please include:



\- a clear description of the vulnerability

\- steps to reproduce it

\- the possible impact

\- any suggested fix, if available



Please do not post security vulnerabilities publicly in issues until the problem has been reviewed.



\## Notes



\- Do not commit real API keys or secrets into the repository

\- Users should provide their own API key through the app

\- Local user data and saved keys should remain on the user's machine
