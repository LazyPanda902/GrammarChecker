# Security Policy

## Supported Versions

This project is currently maintained in its latest public version.

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly to the repository owner.

Please do not post working exploits or sensitive details in public issues.

When reporting a vulnerability, include:

- A short description of the issue
- Steps to reproduce it
- The possible impact
- Any suggested fix, if available

## Secrets

This project does not include a built-in API key.

Each user must provide their own Gemini API key inside the app settings. Real API keys, local settings files, and other private credentials should never be committed to the repository.
